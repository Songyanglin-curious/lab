# 学习系统

## 学习目标

- 明确本套 C# 学习的目标、资料依据、推进规则和完成标准。
- 确认后续所有知识点都按“一次一个点”的方式推进。

当前学习 C# 的核心目标是：系统性学习和梳理 C# 语言本身，建立完整、清晰、可复盘的语言知识体系。面试准备和后续学习 ASP.NET Core 是附带收益，不是当前阶段的最终目标。

当前阶段只聚焦 C# 语言本身，不提前展开 ASP.NET Core、EF Core、数据库、部署、微服务等框架和工程主题。

## 学习规则

- 一次只学一个知识点，不并行跳跃。
- 每个知识点先讲概念，再看代码，再做检查。
- 没有说清楚之前，不勾选完成。
- 遇到框架相关内容，只说明边界，不展开。
- 每个点最后沉淀成自己的理解，而不是只复制资料。
- 可以随时追问“为什么”“和另一个概念有什么区别”“实际项目怎么用”。

## 资料来源

- 普通知识点优先看 Microsoft Learn C# 文档和 C# 语言参考。
- 涉及语法精确定义、边界或争议点时，以 C# 语言规范为准。
- API 行为以 .NET API 文档为准。
- 新版本语法以官方版本说明为准。
- 非官方博客、视频、问答只能作为辅助理解，不能作为最终依据。

## 完成标准

- 能说明为什么要先建立清单和知识点目录。
- 能说明什么情况下可以勾选一个知识点。
- 能说明为什么当前阶段先学纯 C#，不混入框架主线。

单个知识点的通过标准：

- 能用自己的话解释核心概念。
- 能说清它解决什么问题。
- 能说清它在 C# 知识体系里的位置。
- 能说清它和相邻概念的区别。
- 能看懂最小代码示例。
- 能写出一个简单示例。
- 能回答检查题。
- 能说出常见误区或使用边界。
- 能说明当前阶段哪些内容先不展开。
- 如适合反编译，能看懂关键编译结果。
- 如果深入卡住，要标记为 `暂缓深入`，并说明“不影响继续学习”的理由。

## 掌握程度标记

- `基础通过`：已经足够支撑后续学习，可以继续下一个知识点。
- `深入掌握`：不仅会用，还理解语法背后的编译结果、运行机制或设计边界。
- `暂缓深入`：当前已经达到后续学习所需，但某些底层细节暂时吃不下，记录下来以后回看。
- `需要返工`：核心概念还没说清，不能继续。

## 反编译学习规则

- 反编译用于辅助理解编译结果、语法糖展开和运行机制，不替代官方文档和语言规范。
- 适合反编译的知识点要结合最小示例观察关键编译结果。
- 不适合反编译的概念点，只做概念、边界和资料依据梳理。

## 沉淀文件

- `过程笔记.md`：保存学习过程中的临时理解、疑问、代码实验、反编译观察草稿和不适合直接写入总结的内容。
- `对话沉淀.md`：保存有价值的解释和纠偏。
- `总结.md`：保存最终复盘结论。
- `检查题.md`：保存检查题和答题记录。

## 沉淀方式

- 学习过程中优先记录到 `过程笔记.md`，允许不完整、未整理、有疑问。
- 有价值的纠偏、关键追问和容易忘的解释，整理到 `对话沉淀.md`。
- 学完后把稳定结论提炼到 `总结.md`，不要把过程流水账全部搬进去。
- 检查题、回答、订正和通过结果记录到 `检查题.md`。
- 深入但暂时学不下去的内容，记录为 `暂缓深入`，并说明为什么不影响继续学习。
- 通过检查后，更新 `学习清单.md` 和 `学习记录.md`。
- 具体模板见 `docs/CSharp学习/知识点沉淀模板.md`。
