# 认识 CSharp 语言

## 一句话理解

C# 是一门运行在 .NET 平台上的通用编程语言；.NET 是平台；ASP.NET Core 是 Web 开发框架。

## 解决什么问题

本知识点解决的是学习边界问题：先分清语言、平台、框架，避免一开始把 C#、.NET、ASP.NET Core 混成一团。

## 在 C# 体系中的位置

这是整个 C# 学习主线的入口。后续学习类型、变量、方法、类、泛型、委托、LINQ、异常、异步等内容时，都需要先知道哪些属于 C# 语言，哪些属于 .NET 平台能力。

## 核心规则

- C# 是语言，负责语法、类型、表达式、语句、控制流等规则。
- .NET 是平台，包含运行时、基础类库、SDK 和工具链。
- ASP.NET Core 是框架，用来组织 Web 应用开发。
- C# 源码通常先编译为 IL 和元数据，再由 .NET 运行时加载执行。
- IL 通常会在运行时通过 JIT 编译为机器码；部分场景也可以使用 AOT 提前编译。
- 框架能力不是语言能力，框架只是基于语言和平台做更高层封装。
- C# 语言版本主要决定编译器能否接受某种源码写法，以及编译器如何生成 IL 和元数据。
- .NET 版本主要决定目标运行时、基础类库和平台 API 能力。
- 默认搭配不是身份相等：`.NET 8` 项目通常默认使用 C# 12，但 `.NET 8` 不等于 C# 12。
- `TargetFramework` 和 `LangVersion` 可以分开控制，但初学阶段优先按默认搭配理解，避免引入不必要的兼容性问题。

## SDK、Runtime、TargetFramework、LangVersion

- .NET SDK：开发工具包，负责创建、还原、编译、构建、测试和发布项目。
- .NET Runtime：运行时，负责运行已经编译好的程序。
- `TargetFramework`：项目声明自己面向哪个 .NET 平台，例如 `net8.0`、`net6.0`。
- `LangVersion`：项目使用哪一版 C# 语言规则，例如 C# 10、C# 12。

SDK 可以理解为开发期工具箱，通常包含 `dotnet` 命令、C# 编译器、MSBuild、项目模板、NuGet 还原能力、发布工具、参考程序集和部分 Runtime。

```text
SDK 负责把源码变成可运行产物。
Runtime 负责把产物跑起来。
```

常见命令边界：

```text
dotnet new      需要 SDK，用于创建项目
dotnet restore  需要 SDK，用于还原 NuGet 依赖
dotnet build    需要 SDK，用于编译项目
dotnet run      需要 SDK，因为它会构建并运行项目
dotnet publish  需要 SDK，用于发布项目
dotnet MyApp.dll 主要需要 Runtime，用于运行已有产物
```

## 版本降级判断

`.NET 8` 项目降到 `.NET 6` 时，不能只改 `TargetFramework`，还要检查：

- C# 语法：是否使用了 C# 11 或 C# 12 才有的语法，如果降到默认 C# 10，需要改成等效旧写法。
- .NET API：是否调用了 .NET 7 或 .NET 8 才有的 API，如果 `net6.0` 不支持，需要替换为旧 API 或自行实现。
- NuGet 包：不是看包版本号是否比 .NET 版本高，而是看包是否声明支持 `net6.0`。
- 部署环境：目标机器是否安装了能运行 `net6.0` 程序的 Runtime。

一般来说升级容易、降级麻烦。升级通常获得更多能力；降级要求放弃或替换已经使用的新能力。

## 最小代码示例

```csharp
Console.WriteLine("Hello, C#");
```

这行代码中：

- `"Hello, C#"` 是 C# 字符串字面量。
- `Console.WriteLine(...)` 是用 C# 调用表达式调用 .NET 基础类库提供的输出能力。

## 反编译观察

暂不适用。本节只需要理解源码、编译、IL、运行时、机器码之间的大致路径。

## 常见误区

- 误把 C# 当成 .NET。
- 误把 .NET 当成一门语言。
- 误把 ASP.NET Core 当成 C# 语言本身。
- 误以为 `Console.WriteLine` 是 C# 语言内置能力。
- 误以为 C# 支持 AOT 后就等同于 C 或 C++ 的运行模型。
- 误把 C# 语言版本和 .NET 版本当成同一个东西。
- 误以为 `.NET 8` 就是 C# 8 或 C# 12。
- 误以为 SDK 就是 Runtime。
- 误以为降级只需要修改 `TargetFramework`。
