# 对话沉淀

## 关键解释

- C# 是语言，.NET 是平台，ASP.NET Core 是基于 C# 和 .NET 的 Web 框架。
- 语言本身规定语法、类型、表达式、语句、控制流等基础规则；输出、文件、网络、数据库、Web 服务等能力通常来自运行平台、类库或框架。
- `Console.WriteLine("Hello")` 中，字符串字面量和调用表达式属于 C# 语法；`Console`、`WriteLine` 来自 .NET 基础类库。
- `.cs` 文件本质上是源码文本，通常由 C# 编译器编译成 IL 和元数据，再由 .NET 运行时执行。
- C#、F#、VB.NET 都可以编译到 IL，因此它们能在 .NET 平台上运行，也可以在一定边界内相互调用。
- C# 语言版本主要约束源码到编译阶段：这份 C# 源码能不能这么写、编译器能不能接受、编译器如何生成 IL 和元数据。
- .NET 版本主要约束目标平台：编译后的程序集能否在目标平台运行，以及能使用哪些运行时、基础类库和平台 API。
- `LangVersion` 表示项目使用哪一版 C# 语言规则；`TargetFramework` 表示项目面向哪个 .NET 平台。
- .NET SDK 是开发期工具箱，负责创建、还原、编译、构建、测试和发布项目；.NET Runtime 是运行期环境，负责运行已经编译好的程序。
- SDK 通常包含 `dotnet` 命令、C# 编译器、MSBuild、项目模板、NuGet 还原能力、发布工具、参考程序集和部分 Runtime。
- `dotnet build`、`dotnet run`、`dotnet publish` 等开发命令需要 SDK；运行已经发布好的 `MyApp.dll` 主要需要 Runtime。

## 纠偏

- 原理解：C# 和 ECMAScript 都是标准的语法规定。
- 修正后：C# 是语言，ECMAScript 是 JavaScript 的标准规范。C# 也有语言规范，但日常说 C# 时指的是语言本身，不等同于“标准”。

- 原理解：.NET 是运行环境。
- 修正后：.NET 不只是运行环境，它是平台，包含运行时、基础类库、SDK、编译和构建工具链等能力。运行时只是 .NET 的一部分。

- 原理解：IL 是 .NET 能够执行的语言。
- 修正后：IL 是一种中间表示。运行时会加载 IL 和元数据，通过 JIT 编译为机器码执行；也可以通过 AOT 提前编译为本机代码。

- 原理解：C# 可以直接生成二进制代码，这有点像 C 或 C++。
- 修正后：现代 .NET 支持 Native AOT 等提前编译方式，可以生成本机可执行文件。但这不等于 C# 的常规模型变成了 C/C++ 模式；C# 仍然受 .NET 类型系统、运行时模型、基础类库和语言规则约束。

- 原理解：.NET 8 也可以搭配 C# 10，反正能兼容。
- 修正后：`TargetFramework` 和 `LangVersion` 可以分开控制，`net8.0` 项目可以显式指定较低的 C# 语言版本。但默认搭配不等于身份相等，`.NET 8` 不是 `C# 12`，只是通常默认使用 C# 12。

- 原理解：NuGet 包一般不能比当前 .NET 版本高。
- 修正后：NuGet 包看的是是否声明支持目标框架，而不是包版本号大小。高版本包可能支持 `net6.0`，低版本包也可能不支持。

- 原理解：SDK 具体是什么不明确。
- 修正后：SDK 是开发工具包，不是单纯运行时；它负责把源码变成可运行产物，Runtime 负责把产物跑起来。

## 追问结论

- 学 C# 时应先建立语言地基，再进入 ASP.NET Core 等框架。
- 框架没有超越语言和平台的边界，它主要提供组织方式、抽象封装和工程约定。
- 类比 JavaScript 时要分清：语言规范、语言实现、运行时环境、平台 API、框架不是同一层。
- `.NET 8` 项目降到 `.NET 6` 时，不能只改 `TargetFramework`。还要检查 C# 语法、.NET API、NuGet 包兼容性和部署环境。
- 降级通常比升级麻烦，因为降级意味着要放弃或替换已经使用的新语法、新 API、新运行时能力和新依赖能力。
- 最简判断：SDK 负责开发和构建，Runtime 负责运行，TargetFramework 是目标平台声明，LangVersion 是语言规则版本。
