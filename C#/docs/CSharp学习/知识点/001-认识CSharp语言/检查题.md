# 检查题

## 题目

1. C#、.NET、ASP.NET Core 分别是什么？
2. 为什么不能说“.NET 是一门语言”？
3. `Console.WriteLine("Hello")` 这行代码里，哪些部分属于 C# 语法，哪些部分来自 .NET？
4. C# 代码从 `.cs` 文件到运行，大致经历哪几步？
5. 为什么不建议一开始就直接学 ASP.NET Core？
6. C# 语言版本是什么意思？它主要影响语法能力，还是主要影响运行环境能力？
7. .NET 8 项目降到 .NET 6 时，为什么不只是改 `TargetFramework`？
8. SDK、Runtime、TargetFramework、LangVersion 分别是什么？

## 回答

1. C# 是语言，只有语言本身时只能规定基础语法、操作、计算等规则；很多对外部环境的能力来自平台或类库。ASP.NET Core 是一套框架，类似前端中的 Vue，对常用能力做框架性封装，提供更高效、更清晰的组织方式。.NET 是运行环境，类似 V8 或 Node.js 提供代码运行环境以及更多与外部环境交流的能力。
2. 不能说 .NET 是语言，就像不能说 V8 或 Node.js 是语言一样；它们是承载代码运行并提供平台能力的环境。
3. `Console.WriteLine` 是 .NET 提供的打印能力；被打印的内容是 C# 语法中定义的字面量。
4. `.cs` 是源文件，本质是文本；经过编译后生成 IL 中间语言，.NET 能执行这种中间表示。C#、F#、VB.NET 都可以生成 IL，所以都能在 .NET 环境运行，理论上也可以相互调用。C# 也有直接生成二进制代码的能力，这一点类似 C 或 C++。
5. 不直接学习框架，是因为核心目标是知其然知其所以然。为了信息明晰，盖房子要先打地基。
6. C# 语言版本主要管 C# 源码到编译器这一段，决定源码能不能这么写、编译器能不能接受、如何生成 IL 和元数据；.NET 版本主要管目标平台、运行时、基础类库和 API 能力。
7. 因为它们是模块，但有依赖，不是简单改个标记就能处理好不同版本差异。要检查高版本 C# 语法是否需要替换为低版本等效写法，要检查高版本 .NET API 是否需要换成低版本支持的写法，要检查其他依赖和包是否支持当前目标框架，还要检查部署环境。
8. Runtime 是运行时，用来运行已经编译好的程序；TargetFramework 是项目声明“我要面向哪个 .NET 平台”；SDK 是开发工具包，用来开发、编译、构建和发布程序；LangVersion 是 C# 源码使用的语言规则版本。

## 订正

- C# 是语言，不等同于“标准”。C# 有语言规范，但日常说 C# 时指语言本身。
- .NET 不只是运行环境，而是平台，包含运行时、基础类库、SDK、编译和构建工具链等。
- IL 不是最终直接由 CPU 执行的机器码。常规情况下，.NET 运行时加载 IL 和元数据，再由 JIT 编译为机器码执行。
- C# 支持 Native AOT 等提前编译方式，但这不等于 C# 的常规执行模型变成 C 或 C++。
- .NET 版本和 C# 语言版本有关联，但不是同一个东西。`.NET 8` 通常默认搭配 C# 12，但 `.NET 8` 不等于 C# 12，也不等于 C# 8。
- `TargetFramework` 和 `LangVersion` 可以分开控制，但初学阶段优先按默认搭配理解。
- NuGet 包兼容性看的是包是否声明支持目标框架，不是简单看包版本号大小。
- SDK 是开发期工具箱，Runtime 是运行期环境。装了 SDK 通常也能运行程序；只装 Runtime 通常不能创建、还原、编译和发布项目。
