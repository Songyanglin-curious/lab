
# C# 系统学习协作规则

本仓库用于系统学习 C# 语言本身。目标不是快速过教程，而是建立一套完整、可复盘、可持续更新的 C# 知识体系。

## 学习目标

- 一次只学习一个知识点。
- 每个知识点必须讲清、问透、练透、沉淀。
- 当前知识点没有通过检查，不进入下一个知识点。
- 避免夹生饭：不留下模糊定义、不跳过关键边界、不只会写代码而说不清原因。
- 优先建立纯 C# 语言体系，再扩展到 .NET、ASP.NET Core、数据库、部署等工程主题。

## 语言和命名要求

- 目录名、文件名、说明、注释、总结、检查题尽量使用中文。
- C# 官方关键字、类型名、API 名称可以保留英文，例如 `class`、`struct`、`object`、`async`、`Task`、`LINQ`。
- 出现英文术语时，必须给出中文解释。
- 不用生硬翻译影响准确性，核心原则是：中文理解优先，官方术语准确。

## 资料依据

- 核心依据优先使用 Microsoft Learn 中文文档。
- 中文文档不完整或翻译可能有歧义时，对照 Microsoft Learn 英文文档。
- 语言细节以 C# language reference 和 C# specification 为最终依据。
- 其他公开资料只能作为辅助解释，不能替代官方资料。

## 每个知识点的讲解结构

每次学习一个知识点时，默认按以下结构展开：

1. 一句话定义
2. 它解决什么问题
3. C# 为什么这样设计
4. 最小代码例子
5. 真实项目中哪里会出现
6. 和 JavaScript 经验的对比
7. 常见误区
8. 必须记住的判断句
9. 检查题
10. 追问、纠偏、最终总结

## 每个知识点的完成标准

一个知识点只有满足以下条件，才算完成：

- 能用一句话说明它是什么。
- 能说明它解决什么问题。
- 能写出最小代码例子。
- 能指出真实项目中哪里会出现。
- 能说出至少三个常见误区或边界。
- 能和 JavaScript 中容易混淆的概念区分开。
- 能答对检查题，并能解释为什么。
- 有价值的对话、总结、错题已经沉淀到对应知识点目录。

## 文件沉淀规则

- `docs/CSharp学习/学习清单.md` 是总进度入口。
- `docs/CSharp学习/知识点/` 下每个目录对应一个知识点。
- 每个知识点目录中的 `说明.md` 记录学习目标、重点和完成标准。
- 学习过程中产生的高价值内容，后续写入 `对话沉淀.md`、`总结.md`、`检查题.md`。
- 勾选清单时必须基于实际理解和检查结果，而不是只看过内容。

## 范围边界

- 当前主线是纯 C#：语法、类型系统、对象模型、泛型、集合、委托、LINQ、异常、资源、异步、特性、反射、高级语言特性。
- `.NET`、`ASP.NET Core`、`EF Core`、部署、框架源码等不是当前主线。
- 但 `string`、`object`、`Task`、`IEnumerable`、`Exception` 等类型虽然来自 .NET 类库，属于理解 C# 语言能力的必要承载，可以在 C# 主线中讲。
