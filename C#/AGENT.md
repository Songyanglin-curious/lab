
# C# 系统学习协作规则

本仓库用于系统学习 C# 语言本身。目标不是快速过教程，而是建立一套完整、可复盘、可持续更新的 C# 知识体系。

## 学习目标

- 一次只学习一个知识点。
- 每个知识点必须讲清、问透、练透、沉淀。
- 当前知识点没有通过检查，不进入下一个知识点。
- 避免夹生饭：不留下模糊定义、不跳过关键边界、不只会写代码而说不清原因。
- 优先建立纯 C# 语言体系，再扩展到 .NET、ASP.NET Core、数据库、部署等工程主题。

## 语言和命名要求

- 目录名、文件名、说明、注释、总结、检查题尽量使用中文。
- C# 官方关键字、类型名、API 名称可以保留英文，例如 `class`、`struct`、`object`、`async`、`Task`、`LINQ`。
- 出现英文术语时，必须给出中文解释。
- 不用生硬翻译影响准确性，核心原则是：中文理解优先，官方术语准确。

## 资料依据

- 核心依据优先使用 Microsoft Learn 中文文档。
- 中文文档不完整或翻译可能有歧义时，对照 Microsoft Learn 英文文档。
- 语言细节以 C# language reference 和 C# specification 为最终依据。
- 其他公开资料只能作为辅助解释，不能替代官方资料。

## 每个知识点的讲解结构

每次学习一个知识点时，默认按以下结构展开：

1. 一句话定义
2. 它解决什么问题
3. C# 为什么这样设计
4. 最小代码例子
5. 真实项目中哪里会出现
6. 和 JavaScript 经验的对比
7. 常见误区
8. 必须记住的判断句
9. 检查题
10. 追问、纠偏、最终总结

## 每个知识点的完成标准

一个知识点只有满足以下条件，才算完成：

- 能用一句话说明它是什么。
- 能说明它解决什么问题。
- 能说清它在 C# 知识体系里的位置。
- 能写出最小代码例子。
- 能指出真实项目中哪里会出现。
- 能区分它和相邻概念、易混概念的边界。
- 能说出至少三个常见误区或边界。
- 能和 JavaScript 中容易混淆的概念区分开。
- 能答对检查题，并能解释为什么。
- 能说明当前阶段哪些相关内容先不展开。
- 如适合反编译，能看懂关键编译结果。
- 有价值的对话、总结、错题已经沉淀到对应知识点目录。

## 掌握程度标记

- `基础通过`：已经足够支撑后续学习，可以继续下一个知识点。
- `深入掌握`：不仅会用，还理解语法背后的编译结果、运行机制或设计边界。
- `暂缓深入`：当前已经达到后续学习所需，但某些底层细节暂时吃不下，记录下来以后回看。
- `需要返工`：核心概念还没说清，不能继续。
- 如果深入卡住但不影响后续学习，标记为 `暂缓深入`，并说明“不影响继续学习”的理由。

## 反编译学习规则

- 反编译用于辅助理解编译结果、语法糖展开和运行机制，不替代官方文档和语言规范。
- 适合反编译的知识点，要结合最小示例观察关键编译结果。
- 不适合反编译的概念点，只做概念、边界和资料依据梳理。

## 文件沉淀规则

- `docs/CSharp学习/学习清单.md` 是总进度入口。
- `docs/CSharp学习/学习记录.md` 记录每次学习的知识点、关键结论、检查结果和后续问题。
- `docs/CSharp学习/知识点/` 下每个目录对应一个知识点。
- 每个知识点目录中的 `说明.md` 记录学习目标、重点和完成标准。
- 学习过程中优先记录到 `过程笔记.md`，允许不完整、未整理、有疑问。
- 有价值的解释、纠偏、关键追问和容易忘的内容，整理到 `对话沉淀.md`。
- 学完后把稳定结论提炼到 `总结.md`，不要把过程流水账全部搬进去。
- 检查题、回答、订正和通过结果记录到 `检查题.md`。
- 深入但暂时学不下去的内容，记录为 `暂缓深入`，并说明为什么不影响继续学习。
- 通过检查后，更新 `学习清单.md` 和 `学习记录.md`。
- 具体模板见 `docs/CSharp学习/知识点沉淀模板.md`。
- 勾选清单时必须基于实际理解和检查结果，而不是只看过内容。

## 范围边界

- 当前主线是纯 C#：语法、类型系统、对象模型、泛型、集合、委托、LINQ、异常、资源、异步、特性、反射、高级语言特性。
- `.NET`、`ASP.NET Core`、`EF Core`、部署、框架源码等不是当前主线。
- 但 `string`、`object`、`Task`、`IEnumerable`、`Exception` 等类型虽然来自 .NET 类库，属于理解 C# 语言能力的必要承载，可以在 C# 主线中讲。
