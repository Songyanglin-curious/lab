104 协议，全称 **IEC 60870-5-104**，是电力系统里常见的**远动/监控通信协议**，主要用于**主站和厂站、RTU、IED、子站系统之间传输遥测、遥信、遥控、遥脉等数据**。它可以理解为：**把 101 协议的应用层搬到 TCP/IP 网络上跑**。

它不是通用业务协议，而是典型的**点表型协议**：

| 类型 | 含义     | 例子                         |
| ---- | -------- | ---------------------------- |
| 遥信 | 状态量   | 开关分/合、通道在线/离线     |
| 遥测 | 数值量   | 电压、电流、温度、CPU 使用率 |
| 遥控 | 控制命令 | 分闸、合闸、复归、启动/停止  |
| 遥脉 | 累计量   | 电量、次数、累计值           |

104 报文大体分两层：

```
TCP
  └─ 104 APDU
       ├─ APCI：104自己的控制头，管启动、确认、序号、心跳
       └─ ASDU：真正的业务数据，里面放遥信/遥测/遥控等点数据
```

APCI 里有三类帧：**I 帧传数据，S 帧做确认，U 帧做启动/停止/测试控制**。标准资料和厂商文档都按这个结构描述 104 的 I/S/U 帧机制。

ASDU 里通常包含这些核心字段：

```
类型标识 TypeId
传送原因 CauseOfTransmission
公共地址 CommonAddress
信息体地址 InformationObjectAddress
值 Value
品质 Quality
时间 TimeTag，可选
```

你现在可以先把 104 理解成一句话：

> **104 是电力行业用 TCP 传输“点表数据”的标准协议，不是直接传业务对象，而是把业务动作和状态映射成一个个点。**

所以你要把 TLCP 换成 104，关键不是先写 socket，而是先回答这个问题：

> **当前 TLCP 里每一种业务消息，能不能映射成 104 的遥信、遥测、遥控、遥脉？**

能映射的，就走 104 点表；不能自然映射的，后面就会很别扭。比如“开启通道”大概率可以强行映射成**遥控命令**，但如果它背后还有复杂参数、流程状态、文件传输、任务编排，那 104 就不天然适合。




**104 通信是怎么启动的**。
