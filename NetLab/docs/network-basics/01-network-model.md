# 网络基础 01 — 网络是什么

> 目标：理解 IP、端口、客户端-服务端模型。这是所有网络通信的"世界地图"。

---

## 1. 问题引入

RemoteOps 的生产代码里，`MainServer` 要给 `ChannelServer` 发管理命令，给 `AgentServer` 发代理请求。它们跑在不同的机器（或同一台机器的不同进程）上，怎么让对方收到？

答案：**网络通信**。而理解网络通信，先要理解三个核心概念。

---

## 2. 三个核心概念

### 2.1 IP 地址 — 找哪台机器

```
类比：快递上的街道地址

  你要寄快递到 "北京市朝阳区XX路XX号"
  IP 地址就是网络世界里的 "门牌号"

  127.0.0.1 = "本机"（自己寄给自己，测试用）
  192.168.1.x = 局域网内某台机器
```

IP 地址是网络层寻址，确定数据去哪台机器。一台机器可以有一个或多个 IP。

### 2.2 端口 — 找哪个程序

```
类比：大楼里的房间号

  快递到了公司大楼（IP），但楼里有 100 个办公室（程序）
  端口告诉快递员：送到 3 楼 305 室

  端口号范围：0 ~ 65535
  同一个端口在同一台机器上只能被一个程序占用
```

| 端口范围 | 用途 |
|----------|------|
| 0~1023 | 系统保留（HTTP=80, HTTPS=443） |
| 1024~49151 | 注册端口 |
| 49152~65535 | 动态/私有端口 |

RemoteOps 的端口分配（来自 `SystemConfig.cs`）：

| 端口 | 服务 |
|------|------|
| 9101 | ChannelServer |
| 9102 | ManageServer |
| 9103 | AgentServer |
| 9104 | OperateServer |
| 9105 | MonitorServer |

### 2.3 客户端-服务端模型

```
┌──────────┐          ┌──────────┐
│  客户端   │ ──连接──> │  服务端   │
│ (Client) │ <──响应── │ (Server) │
└──────────┘          └──────────┘

  服务端：启动后一直等待别人连过来（被动）
  客户端：知道服务端地址，主动发起连接（主动）
```

关键区别：

| 维度 | 客户端 | 服务端 |
|------|--------|--------|
| 谁先启动 | 后启动 | 先启动（监听端口） |
| 知道谁的地址 | 知道服务端的 IP:Port | 不知道谁会连过来 |
| 发起连接 | 主动 | 被动（accept） |
| 一个端口 | 不占用固定端口 | 占用固定端口 |

## 3. 用餐厅类比理解

```
  餐厅 = 服务端（Server）
    - 地址：XX路XX号 = IP 地址
    - 桌号：3号桌 = 端口号
    - 服务员在门口等客人 = 监听（Listen）

  客人 = 客户端（Client）
    - 知道餐厅地址 = 知道 IP
    - 走进餐厅坐到 3 号桌 = 连接（Connect）

  一个餐厅可以有多个客人同时在吃饭 = 一个服务端可以同时服务多个客户端
```

---

## 4. 和 RemoteOps 的关系

RemoteOps 里每个服务都是**既做服务端又做客户端**：

```
MainServer（主站）
  - 作为客户端 → 连 ChannelServer:9101（查通道状态）
  - 作为客户端 → 连 AgentServer:9103（发代理命令）

ChannelServer（通信网关）
  - 作为服务端 ← 监听 9101（等 MainServer 连过来）
  - 作为客户端 → 连 AgentServer:9103（转发管理命令）

AgentServer（变电站代理）
  - 作为服务端 ← 监听 9103（等别人连过来）
```

中间的箭头就是 `Net.InternalSend("ip", port, message)` 做的事。

## 5. 常见坑

| 坑 | 解释 |
|----|------|
| 端口被占用 | 两个程序不能监听同一个端口。`Address already in use` |
| 连 127.0.0.1 和连真实 IP 不一样 | 127.0.0.1 走回环接口不经过网卡；真实 IP 走网卡 |
| 防火墙挡了 | 服务端端口被防火墙拦了，客户端连过去没反应 |
| 客户端端口随机 | 客户端发起连接时，OS 自动分配一个空闲端口，不用手动指定 |
